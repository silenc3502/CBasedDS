## Hash

```make
해시 함수는 정의 및 구현에 따라 일반적으로 PRNG (Pseudo Random Number Generators)로 간주된다.
이 일반화에서 해시 함수의 성능과 다른 해시 함수 간의 비교는 함수를 PRNG로 모델링하여 결정될 수 있다고 가정 할 수 있다.
푸아송 분포와 같은 분석 기술을 사용하여 서로 다른 데이터 그룹에 대한 서로 다른 해시 함수의 충돌률을 분석 할 수 있다.
일반적으로 특정 데이터 그룹에 대해 Perfect Hash Function으로 알려진 이론적 해시 함수가 있다.
정의에 따라 완벽한 해시 함수는 충돌이 발생하지 않음을 의미하며
이는 그룹의 다른 요소에서 반복되는 해시 값이 발생하지 않음을 의미한다.
실제로 임의의 데이터 세트에 대한 완벽한 해시 함수를 찾는 것은 매우 어렵고,
또한 완벽한 해싱 및 변형 최소 완벽한 해싱의 실제 응용 프로그램은 상당히 제한적이다.
따라서 대신 특정 데이터 세트에 대해 가장 적은 양의 충돌을 생성하는 함수로
일반적으로 정의되는 이상적인 해시 함수의 개념을 추구하도록 한다.
데이터를 해싱하거나 특정 도메인에서 다른 도메인으로 값을 매핑하는데 있어
근본적인 문제 중 하나는 데이터 유형의 순열이 너무 많다는 것이다.
일부는 매우 무작위적이며 다른 일부는 높은 수준의 패턴화 또는 구조를 포함하여 해시를 일반화하기 어렵다.
모든 데이터 유형 또는 특정 데이터 유형에 대한 기능, 누구나 할 수 있는 일은 시행 착오를 통해
공식 해시 함수 구성 기술을 활용하여 필요에 가장 적합한 해시 함수를 도출하는 것이다.

```

## Data Distribution

```make
이것은 해시 함수가 데이터 집합 내에서 요소의 해시 값을 얼마나 잘 배포하는지 측정한 것이다.
이 측정에서 분석하려면 고유하지 않은 해시 값을 의미하는 데이터 세트에서 발생하는 충돌 수를 알아야한다.
충돌 해결을 위해 체인이 사용되는 경우 체인의 평균 길이 (이론적으로는 각 버킷의 충돌 횟수 평균)
또한 범위 내 해시 값 그룹화의 양도 분석해야한다.
```
